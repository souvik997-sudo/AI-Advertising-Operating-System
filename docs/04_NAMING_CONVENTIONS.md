# Naming Conventions

## Purpose

This document defines the naming standards used throughout the AI Advertising Operating System (AAOS).

Consistent naming improves readability, maintainability, and collaboration.

---

# File Naming

Use:

NN_Module_Name.md

Examples:

01_Master_System.md

02_System_Architecture.md

03_AI_Identity.md

14_Script_Framework.md

20_Checklists.md

---

# Folder Naming

Use lowercase names.

Examples:

assets/

clients/

docs/

examples/

modules/

prompts/

templates/

---

# Markdown Files

Use:

Pascal_Case_With_Underscores.md

Correct:

Buyer_Psychology.md

Brand_Analysis.md

Wrong:

buyerpsychology.md

buyer-psychology.md

Buyer Psychology.md

---

# Prompt Files

Use descriptive names.

Examples:

facebook_ad.md

seo_article.md

landing_page.md

ugc_script.md

---

# Example Files

Use:

example_name.md

Examples:

restaurant_ad.md

clinic_landing_page.md

seo_blog.md

---

# Assets

Images:

logo.png

banner.jpg

workflow.png

---

# Versioning

Major updates:

v1.0

v2.0

v3.0

Minor improvements:

v1.1

v1.2

v1.3

---

# Branch Naming

feature/new-module

feature/seo-engine

feature/brand-analysis

fix/readme

docs/update

---

# Commit Messages

Use short and meaningful messages.

Examples:

Add SEO framework

Improve README

Fix naming conventions

Update documentation

Refactor project structure

---

# General Rules

- Use English only.
- Avoid spaces in filenames.
- Prefer underscores for module names.
- Keep names descriptive.
- Maintain consistency across the project.

---

# Future Rule

All new modules, prompts, templates, and examples should follow these conventions.
